# Subagent Workflow Patterns

This document covers execution strategies, context management rules, foreground vs. background execution, subagent resuming, and compaction behaviors for subagents in Claude Code.

---

## 1. Invocation Models

Claude Code supports three ways to invoke a subagent, ranging from a one-off request to a project-wide default.

### A. Automatic Delegation (Natural Language)
Claude automatically decides to spawn a subagent when it encounters a self-contained task matching the subagent's `description`.
* *Tip*: Include instructions like "use proactively after code edits" in the description to encourage self-delegation.
* *Example*: `Use the code-reviewer subagent to look at my recent changes.`

### B. Explicit @-Mention (Guaranteed Spawn)
Force a specific subagent to take on the next prompt.
* *Syntax*: Type `@` inside the prompt and select the subagent, or write it manually:
  * Local/User/Project: `@agent-code-reviewer look at the auth changes`
  * Plugin subagent: `@agent-my-plugin:code-reviewer review these specs`

### C. Session-Wide Default (Main Thread Override)
Set the entire session to take on the subagent's system prompt, model, and tool restrictions.
* *CLI override*: `claude --agent code-reviewer`
* *Persistent Setting* (in `.claude/settings.json`):
  ```json
  {
    "agent": "code-reviewer"
  }
  ```

## 2. Foreground vs. Background Execution

Subagents can run in either of two concurrency modes, determined by the host or command context:

```yaml
concurrency_modes:
  foreground:
    blocking: true
    prompts: "Permission prompts are passed through to the terminal dynamically."
    clarifications: "Allows active interactive back-and-forth."

  background:
    blocking: false
    prompts: "Runs silently and concurrently. Auto-denies any tool call that would trigger a permission prompt."
    clarifications: "Clarifying question calls will fail, but the agent attempts to bypass or proceed."
    manual_triggers:
      - "Add 'run this in the background' in your natural language prompt."
      - "Press 'Ctrl+B' to push a running foreground subagent into the background."
    disable_all: "Set environment variable `CLAUDE_CODE_DISABLE_BACKGROUND_TASKS=1` to enforce synchronous tasks only."
```

## 3. High-Fidelity Design Patterns

### Pattern A: Isolate High-Volume Operations
Verbose console outputs (running tests, large file lookups, heavy log parsing) pollute the main thread's context history, leading to fast prompt cost escalation.
* **Practice**: Always route test-running or heavy log-grepping to a custom subagent. The subagent digests the massive output and returns only a clean, actionable summary.

### Pattern B: Parallelized Research
If you need to analyze multiple independent codebase directories (e.g. `auth`, `database`, `api`), delegate each directory to a separate background subagent.
* **Practice**: The subagents explore isolated folders concurrently, and the main Sonnet session synthesizes their returned summaries.

### Pattern C: Pipeline Chaining
Sequence multiple specialists to refine output step-by-step.
* **Practice**:
  1. Trigger `@agent-code-reviewer` to audit a file and output issues.
  2. Trigger `@agent-debugger` to implement fixes based on the reviewer's checklist.

## 4. Context Boundary Rules

Each non-fork subagent starts with a **fresh, completely isolated context window**. It has no visibility into the main session's previous history, loaded files, or active memory unless explicitly passed.

```yaml
context_window_payload:
  system_prompt: "The subagent's markdown body prompt + basic environment details (CWD)."
  task_message: "The delegation prompt generated by the parent agent."
  global_configs: "CLAUDE.md, CLAUDE.local.md, and global managed settings. (Exception: Explore and Plan built-in subagents skip these)."
  git_status: "Snapshot of active workspace changes (Exception: Explore and Plan skip this)."
  preloaded_skills: "Fully resolved content of skills listed in the `skills` frontmatter field."
```

## 5. Resuming Stopped Subagents

Every new agentic call creates a new instance. To keep working with an already configured, running, or suspended subagent (retaining its tool history and reasoning), you can resume it.

* **Under the Hood**:
  * Transcripts are stored locally under `~/.claude/projects/{project}/{sessionId}/subagents/` as `agent-{agentId}.jsonl`.
  * Subagents are unaffected by main conversation history compaction.
  * To resume, Claude uses the `SendMessage` tool with the target subagent's `agentId` in the `to` field.
  * *Requirement*: Requires the experimental agent teams setting:
    `CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS=1`

---

## 6. Auto-Compaction Rules

To handle long agentic tasks, subagents support automatic transcript compaction.

```yaml
compaction_policy:
  trigger_threshold: "Approximately 95% of model context window capacity."
  override_variable: "CLAUDE_AUTOCOMPACT_PCT_OVERRIDE"
  custom_threshold: "For example, set to 50 to force compaction when half full."
```
